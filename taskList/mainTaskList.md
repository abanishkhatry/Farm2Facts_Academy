# Task List

This page is the central index of all student tasks across the 8-week program. Each week links to a dedicated markdown file with detailed step-by-step instructions, acceptance criteria, and guidance for completing every task in that week. Work through the weeks in order, following the issue-branch-PR workflow for each task.

---

- **Week 1 -- Environment Setup and Orientation:** Get both the frontend and backend repositories running locally, verify your development environment, and orient yourself to the codebase structure before making any changes.

- **Week 2 -- First Contribution:** Make your first real change to the codebase using the full issue-branch-PR workflow. Tasks are kept small in scope so the focus is on the process, not the complexity.

- **Week 3 -- Data Entry and Forms:** Work inside the instrument forms that collect farmers market data. Add validation to an existing form field and extend an instrument with a new field.

- **Week 4 -- API Integration:** Work across the frontend-backend boundary. Trace an existing API call end to end and extend a backend endpoint with a new query parameter.

- **Week 5 -- Code Quality and Testing:** Introduce linting and automated tests to the repos. Set up a pre-commit hook and write tests for an existing backend endpoint.

- **Week 6 -- Performance and Data Display:** Improve how data is loaded and shown. Add server-side pagination to a data table and profile a slow API endpoint to identify its bottleneck.

- **Week 7 -- Feature Development:** Build something new from issue through to merged PR. Add a summary stat to a dashboard view and implement a new Chart.js visualization.

- **Week 8 -- Independent Work and Wrap-Up:** Choose your own contribution from the issue backlog, complete the full workflow independently, and document one design decision as an ADR.
